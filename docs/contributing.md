# Contributing to Aero

Thanks for considering a contribution.

## Workflow

1. Fork and branch from `develop`.
2. Follow the conventions in [developer-guide.md](developer-guide.md).
3. Run `npm run lint && npm run typecheck` before pushing.
4. Open a PR against `develop` with a clear description and screenshots/GIFs for UI changes.

## Commit style

Use clear, present-tense commit messages, e.g. `Add Lost Mode alarm trigger`, `Fix battery tile rounding`.

## Reporting issues

Please include: Android version, device model, steps to reproduce, and expected vs. actual behavior.
